# Инструкция: массовый анализ travel-фотографий с помощью параллельных суб-агентов

Этот документ описывает проверенный процесс создания рекомендательных документов по фотографиям из путешествий. Процесс отработан на ~3000 фотографиях из трёх поездок (Турция 2024, Турция 2025, Тайланд 2025).

## 1. Цель

Создать Markdown-файл `recommendations-<название-транслитом>.md`, в котором каждая фотография описана так, чтобы человек мог выбрать лучшие снимки **только по текстовому описанию**, без повторного просмотра. Фотографии группируются по сценам, для каждой группы выбирается лучший кадр.

## 2. Структура итогового файла

```markdown
# Рекомендации по фотографиям: <Название поездки>

<Краткое описание: что за поездка, даты, цель документа>

> Время в названии файлов Pixel записано в UTC. Местное время пересчитано с поправкой +N часов (GMT+N, <страна>).

## Группа 1: <Название сцены>

| Файл | Местное время (GMT+N) | Рекомендация | Причина | Описание |
|------|----------------------|--------------|---------|----------|
| PXL_... | дд.мм.гггг чч:мм | ✅ Лучший | ... | 5-7 предложений |
```

## 3. Подготовка изображений

Оригиналы с Google Pixel весят 5-10 МБ каждый (12+ мегапикселей). Суб-агенты не смогут эффективно работать с такими файлами — контекст переполнится, а расход токенов будет огромным. Поэтому перед анализом нужно создать миниатюры.

### Скрипт `compress.py`

В корне проекта лежит скрипт `compress.py`, который создаёт сжатые копии изображений. Он:
- Уменьшает длинную сторону до заданного размера (по умолчанию 640px)
- Сохраняет в JPEG с заданным качеством (по умолчанию 40)
- Сохраняет структуру подпапок
- Выходную папку создаёт рядом с собой (в рабочей директории), с тем же именем что и исходная (или с именем из `--output-name`)
- Поддерживает форматы: jpg, jpeg, png, gif, bmp, tiff, webp
- Требует `Pillow` (`pip install Pillow`)

### Использование

```bash
# Дефолтные параметры (640px, quality 40) — проверены на Турции 2025, работают хорошо
python compress.py "D:/Media/Photo archive/2025/Турция 2025"

# С кастомными параметрами
python compress.py "D:/Media/Photo archive/2025/Турция 2025" --max-size 800 --quality 50

# С кастомным именем выходной папки
python compress.py "D:/Media/Photo archive/2025/Турция 2025" --output-name "Турция 2025 v3"
```

### Параметры

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `source` (позиционный) | — | Путь к папке с оригиналами |
| `--max-size` | 640 | Максимальная сторона в пикселях. При 640px оригинал ~4000x3000 уменьшится до ~640x480 |
| `--quality` | 40 | Качество JPEG (1-100). 40 даёт файлы ~30-80 КБ вместо 5-10 МБ. Для ИИ-анализа этого достаточно |
| `--output-name` | имя исходной папки | Имя выходной папки. Полезно для создания нескольких версий (v2, v3) |

### Проверенные параметры

Для проекта Турция 2025 использовались **дефолтные параметры** (`--max-size 640 --quality 40`). Результат:
- Оригинал: ~5-10 МБ → миниатюра: ~30-80 КБ (сжатие в 100-200 раз)
- Суб-агенты Haiku успешно читали и описывали все изображения
- Достаточно деталей для определения лиц, поз, одежды, пейзажей
- 76 миниатюр = ~3-5 МБ суммарно, что укладывается в контекст одного агента

### Важно

- Скрипт НЕ трогает оригиналы — создаёт копии в отдельной папке
- Выходная папка создаётся в текущей рабочей директории (там, откуда запускается скрипт)
- Если выходная папка уже существует — файлы перезаписываются без предупреждения

## 4. Разбивка на чанки

Раздели список файлов на равные части (~75 файлов на суб-агент):

```bash
ls "папка_с_миниатюрами/" | sort > filelist.txt
split -l 76 filelist.txt chunk_
```

Это создаст файлы `chunk_aa`, `chunk_ab`, ..., `chunk_at` и т.д. Количество чанков = количество суб-агентов. Оптимально 15-25 агентов.

## 5. Промпт для суб-агентов

Промпт хранится в файле `agent_prompt_template.md` (в этой же директории). **Не пересказывай и не переписывай его** — дай суб-агенту команду прочитать этот файл самостоятельно. Это критически важно: при пересказе теряются детали и нюансы.

### Что содержит промпт:
- Инструкцию по чтению изображений через Read tool
- Запрет на угадывание содержимого по имени файла
- Объяснение, что файлы — миниатюры, и артефакты сжатия не являются дефектами фото
- 5-шаговый алгоритм описания (композиция → фокус → эмоции → свет → контекст дубля)
- Примеры хороших и плохих описаний
- Запрет на пустые оценки ("красивый пейзаж", "хорошее освещение")
- Формат таблицы и группировки по сценам

### Переменные в промпте:
- `{CHUNK_SUFFIX}` — суффикс файла чанка (aa, ab, ac, ...)
- `{BATCH_NUM}` — номер батча (01, 02, 03, ...)
- Путь к папке с изображениями — указан в промпте, менять при необходимости
- Часовой пояс — указан в промпте, менять при необходимости

## 6. Запуск суб-агентов

### Команда запуска (для каждого суб-агента):

```
Agent(
  model="haiku",
  prompt="Прочитай файл <путь>/agent_prompt_template.md — это твоё задание. Следуй ему ДОСЛОВНО.\n\nПодставь переменные:\n- {CHUNK_SUFFIX} = <суффикс>\n- {BATCH_NUM} = <номер>\n\nНачинай работу.",
  run_in_background=true
)
```

### Критические правила запуска:

1. **Запускай ВСЕ суб-агенты одновременно** в одном сообщении. Не жди завершения одного перед запуском другого.

2. **Модель: haiku**. Она дешевле по токенам и справляется с задачей при правильном промпте. Sonnet/Opus дороже, но дают более качественные описания — используй их если бюджет позволяет.

3. **НЕ пересказывай промпт** суб-агенту. Дай ему прочитать файл `agent_prompt_template.md` самостоятельно. Любой пересказ искажает инструкции.

4. **Промпт для суб-агента должен быть минимальным**: прочитай файл + подставь переменные + начинай. Всё остальное — в файле промпта.

## 7. Мониторинг и проверка

### После завершения всех агентов проверь:

```bash
# Проверить что все батчи созданы и содержат ожидаемое количество файлов
for i in $(seq -w 1 20); do
  f="temp_t25v3_batch_${i}.md"
  files=$(grep -o 'PXL_[0-9_]*[A-Z.]*jpg' "$f" | sort -u | wc -l)
  echo "Batch $i: ${files} files"
done
```

### Красные флаги (нужен перезапуск):
- **0 файлов** — агент не читал изображения (заявил что "не умеет")
- **Менее 50% от ожидаемого** — агент остановился на полпути (лимит контекста/токенов)
- **tool_uses < количество файлов** — агент не читал каждый файл через Read

### При перезапуске отдельных батчей:
- Проверь, на каком файле агент остановился (последний PXL_* в файле)
- Создай новый чанк-файл только с необработанными файлами
- Запусти нового агента с этим чанком, указав дописывать в существующий файл (Bash append, не Write)

## 8. Сборка финального файла

После того как все батчи готовы:

```bash
# Создай заголовок
cat > recommendations-turkey-2025.md << 'EOF'
# Рекомендации по фотографиям: Турция 2025
...заголовок и описание...
EOF

# Добавь все батчи, убрав их индивидуальные заголовки
for i in $(seq -w 1 20); do
  sed '/^# Батч/d' "temp_t25v3_batch_${i}.md" >> recommendations-turkey-2025.md
  echo "" >> recommendations-turkey-2025.md
done
```

### Перенумерация групп (если нужна сквозная нумерация):

```python
import re
with open("recommendations.md", encoding="utf-8") as f:
    content = f.read()
counter = [0]
def repl(m):
    counter[0] += 1
    return f"## Группа {counter[0]}: {m.group(1)}"
result = re.sub(r"^## Группа \d+: (.+)$", repl, content, flags=re.MULTILINE)
with open("recommendations.md", "w", encoding="utf-8") as f:
    f.write(result)
```

## 10. Уроки из практики (что шло не так)

### Проблема: Haiku отказывается читать изображения
**Симптом:** Агент пишет "я не могу визуально анализировать изображения" или "requires vision API".
**Причина:** Модель не знает, что Read tool умеет показывать JPG.
**Решение:** В промпте явно написано: "Read tool с путём к JPG = ты УВИДИШЬ картинку. Это ПРОВЕРЕННЫЙ ФАКТ." Не убирай эту строку.

### Проблема: Агент придумывает описания по именам файлов
**Симптом:** tool_uses << количества файлов, описания шаблонные.
**Причина:** Агент решил сэкономить и не читать файлы.
**Решение:** В промпте есть строгий запрет: "БЕЗ вызова Read писать описание КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО."

### Проблема: Агент пишет "размытый кадр", "кадр из видео"
**Симптом:** Описания содержат жалобы на качество.
**Причина:** Агент видит артефакты миниатюры и принимает их за дефекты фото.
**Решение:** В промпте есть блок про миниатюры. Файлы .MP.jpg — это Motion Photo Google Pixel (полноценная фотография), не кадр из видео.

### Проблема: Агент обрабатывает только часть файлов
**Симптом:** В файле 15-30 записей вместо 75.
**Причина:** Контекст переполнился или агент "забыл" продолжать.
**Решение:** Инструкция "каждые 15 файлов — сразу пиши в файл" решает проблему потери данных. Если агент всё равно остановился — перезапусти с оставшимися файлами.

### Проблема: Скудные описания ("портрет, хорошее освещение")
**Симптом:** Описания в 3-5 слов вместо 3-5 предложений.
**Причина:** Недостаточно строгий промпт или слабая модель.
**Решение:** Промпт содержит 5-шаговый алгоритм с примерами хороших и плохих описаний, а также строжайшие табу на пустые оценки. Не сокращай эту часть промпта.

### Проблема: Rate limit / лимит токенов
**Симптом:** Агент возвращает "You've hit your limit".
**Причина:** Слишком много параллельных агентов или слишком тяжёлые изображения.
**Решение:** Уменьши размер миниатюр, уменьши количество параллельных агентов, или подожди сброса лимита.

## 11. Конфигурация под другие поездки

Для новой поездки измени в `agent_prompt_template.md`:
1. Путь к папке с изображениями
2. Часовой пояс (GMT+N)
3. Формат имени файла (если не Pixel — адаптируй парсинг времени)

Всё остальное (алгоритм описания, формат таблицы, правила группировки) — универсально.
